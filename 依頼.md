次に、Super Adminの配布導線が本番デプロイでも成立するか確認・整備してください。

前提:

* macOS版「予約同期くん」のdmgはApple公証済みです。
* Worker側のdmgは `./KIREIDOT_Salonboard_Worker/release/salonboard-sync-mac-arm64.dmg`
* Super Admin側の配布先は `./KIREIDOT_Super_Admin/public/downloads/salonboard-sync-mac.dmg`
* SHA256は `./KIREIDOT_Super_Admin/public/downloads/salonboard-sync-mac.dmg.sha256`
* ただし、dmg本体はgitignoreされている可能性があります。

やってほしいこと:

1. Super AdminのGit管理状況を確認してください。

   * `public/downloads/salonboard-sync-mac.dmg` がgitignoreされているか確認してください。
   * `public/downloads/salonboard-sync-mac.dmg.sha256` がコミット対象になっているか確認してください。
   * `src/app/downloads/page.tsx` と `public/downloads/README.md` の変更状態を確認してください。

2. 本番デプロイ時にdmgが配布される構成になっているか判定してください。

   * Git経由のVercel/Netlify等のデプロイで、gitignoreされたdmgが本番に含まれるか確認してください。
   * 含まれない場合は、以下のどちらかの方針を提案してください。
     A. dmgを外部ストレージに置き、Super Adminから外部URLへリンクする
     B. 一時的にdmgをGit管理対象にする
   * 100MB前後のdmgなので、Git管理は推奨しない方針でお願いします。

3. 推奨方針は、外部ストレージ配布にしてください。
   候補:

   * GitHub Releases
   * Cloudflare R2
   * S3
   * Vercel Blob

4. 今回はまず最小変更で、Super Admin側を「外部ダウンロードURL対応」にしてください。

   * `.env` または環境変数で `NEXT_PUBLIC_SALONBOARD_SYNC_MAC_DMG_URL` を受け取れるようにしてください。
   * URLが設定されている場合は、そのURLをダウンロードボタンに使ってください。
   * URLが未設定の場合は、従来通り `/downloads/salonboard-sync-mac.dmg` を使ってください。
   * SHA256表示は引き続き `.sha256` から読めるようにしてください。
   * 既存UIは壊さないでください。

5. `public/downloads/README.md` を更新してください。

   * dmg本体はGitに入れず、外部ストレージに置く方針を明記してください。
   * SHA256ファイルはGit管理する方針を明記してください。
   * 新しいdmgを配布する手順を書いてください。
   * 公証済みdmgをアップロード → URLを環境変数に設定 → SHA256更新 → デプロイ、という流れにしてください。

6. 最後に以下を報告してください。

   * Git管理されているファイル一覧
   * gitignoreされているファイル一覧
   * 本番でdmgが404にならないための方針
   * 追加・変更した環境変数
   * 次に人間がやるべきこと
   * コミットすべきファイル一覧

注意:

* dmg本体を勝手にGitへ追加しないでください。
* Apple ID、App用パスワード、証明書、秘密鍵は触らないでください。
* 既存の公証済みdmgを削除しないでください。
