SalonBoard連携 機能

マルチテナント 配布・運用フロー（2026-06-13）

同一の worker を各店舗PCに配布し、店舗の実回線（住宅IP）で SalonBoard を操作。頭脳（クラウド）が全店を監視・協調する。

💻 予約同期くん（Electron Worker）

  • 初回のみ手動インストール（各店舗PCで .dmg を実行）
  • 以降は electron-updater が 起動時/定期に自動更新
  • そのPCの実回線で SalonBoard を操作 → Akamai を通る
⚠️ 「各店舗で自動インストールするスクリプト」は無し。初回は各PCで手動セットアップが必要（その後は自動更新）。

4. なぜ「各店舗PC」なのか
❌ AWS / さくら / カゴヤ等（データセンターIP）
Akamai が「DCからのブラウザ」をボットとして遮断。実機検証で全パターン不可。
→
✅ 各店舗の実回線（住宅/事業者IP）
「その店のスタッフが普段どおり自店のSalonBoardを使う」のと同一 → 検知リスク最小・追加コストなし。

👁 監視・見える化
