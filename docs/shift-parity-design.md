# シフト整合(KD↔SB Shift Parity)設計 — 2026-07-17

## 目的
**全店で「KIREIDOTのシフト時間」と「SalonBoardのシフト時間」を確実に一致させる。**
→ 予約が `受付可能数超過/予定重複` で失敗する根本(SBシフト時間ズレ)を根絶し、予約同期を100%化する。

## 今日判明した構造的欠陥(実測)
1. **勤務パターン紐付けの空白**: `salonboard_shift_patterns.matched_preset_id` が全店ほぼ未設定。→ push_shifts が KDシフト時間(平日遅番12:30-21等)を SB勤務パターンに変換できない。
2. **時間を見ない差分**: push_shifts の差分は **セル表示(パターン短縮名/出/休)** だけ比較(`cellMatchesPattern`)。**実時間(セルクリックで出る時刻)を見ない** → SBが 10:00-11:00+19:30-21:00 でも「出勤」なら KD 12:30-21:00 と「一致」誤認 → 永久に直さない。
3. **予定方式フォールバックの精度**: パターン紐付けが無い日は「出勤+予定(時間外ブロック)」方式に落ちるが、ベースパターン近似で **全く違う時間が入る**(実例: Seri 代々木 SB=10-11+19:30-21)。
4. 結果: SBシフト≠KD → 予約枠にスタッフ不在扱い → 登録失敗。

## 設計原則
- **KDプリセット(`organizations.shift_presets`)が唯一の真実(source of truth)**。SBはそのミラー。
- **1 KDプリセット ⇔ 1 SB勤務パターン(同一時刻)** を全店で保証する。
- **手動作業をゼロに**(現状の「手動で紐付け」が空白の温床)。自動マッチ+自動登録+自動検証。

## コンポーネント

### ① パターン台帳の整備(基盤)
- **fetch_shift_patterns**: SB登録の勤務パターン(名前+時刻)を `salonboard_shift_patterns` へ吸い上げ(既存)。
- **自動マッチ(新規)**: 取得後、SBパターンとKDプリセットを **開始・終了時刻の完全一致**で自動 `matched_preset_id` 設定。手動紐付けを廃止。→ 空白の根絶。
- **不足パターンの自動登録(新規/既存流用)**: KDプリセットに対応するSB勤務パターンが無ければ、`push_shift_patterns`(既存 pushWorkPatternViaForm)で **SBに勤務パターンを登録**(同名・同時刻)。→ 全プリセットにSB対応を用意。
- **未マッチ検知**: 一致しないSBパターン/未登録プリセットは Slack 警告(要手動確認)。

### ② push_shifts を「時刻正確」に(中核修正 = 要worker改修)
- **紐付け前提化**: ①で全KDシフトが matched パターンに解決 → **予定方式(近似)を原則使わない**(最後の手段のみ、使用時は必ず Slack 明細)。
- **差分を実時刻ベースに**: セル表示名だけでなく、**設定したパターンの時刻がKDと一致するか**で判定。パターン紐付けが正しければ「名前一致=時刻一致」が保証される(①が前提)。
- **反映後リードバック検証**: 一括入力後、対象セルの実際の値がKD期待どおりか確認(現状は先頭セルのみ確認)。ズレたら再設定。

### ③ 継続的ドリフト検知(再発防止)
- **verify_shifts(新規 or fetch拡張)**: SBの実シフト時刻を定期取得し KDと突合。ズレを検知したら **自動 re-push + Slack 通知**。→ 「予約が失敗して初めて気づく」を廃止し、予約前にシフトを整合させる。
- **予約前ガード**: push_booking 実行前に「担当のSBシフトが予約時間をカバーするか」を(同期済みシフト状態で)チェック。カバー外なら **cryptic な受付可能数超過ではなく「シフト未整合」で明示 alert**＋manual_required。

### ④ ロールアウト & 運用
- パイプライン(店単位・冪等): `fetch_shift_patterns → 自動マッチ → 不足を自動登録 → push_shifts(全スタッフ月次) → verify`。
- 順序: **確定破損の高稼働店(代々木・新宿)で検証 → 全Unelimit → 全店**。
- 監視: 未マッチ/ドリフト/未登録プリセットを Slack #kireidot-info に常時。
- **スタッフ紐付け(`salonboard_staff_imports.matched_staff_id`)の完全性**も前提(別だが関連。未紐付けスタッフのシフトは反映されない)。

## データモデル
- `organizations.shift_presets` — KD正(canonical presets)。
- `salonboard_shift_patterns(shop_id, external_id, name, start_time, end_time, matched_preset_id)` — SBパターン+KD紐付け(時刻で自動マッチ)。
- 追加候補: 店別 `shift_parity`(matched数/unmatched数/last_verified_at)で健全性を可視化。

## 完了の定義
- 任意店・任意スタッフで **SBシフト時刻 == KDシフト時刻**(verifyで0ドリフト)。
- manual_required にした予約を再enqueue → **登録成功**。
- 以後、シフト変更が **予約失敗を起こさず** SBへ正しく反映。
